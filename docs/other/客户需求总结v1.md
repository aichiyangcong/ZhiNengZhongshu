新 demo 不该只是“更好看的看板”，而应该是“经营智能
  中枢 + 任务闭环引擎”的产品化演示，核心是 发现问题 -> 解释原因 ->
  生成动作 -> 跟踪结果。

  我建议的新产品 demo 方向（在你旧 demo 基础上升级）

  - 主线定位：AI经营中枢 2.0，强调“闭环”而非“报表”
  - 统一框架（三类客户都吃得下）：
      - 数据层：多源数据归口（POS/外卖/督导/供应链/财务/外部因子）
      - 决策层：Top Issues + 根因树 + 风险评分
      - 执行层：自动派单/催办/升级/复盘沉淀SOP
  - 角色视角：总部运营总、区域督导、店长（必要时加“服务商经理”）

  ———

  从 3 份访谈抽出来的共性需求（一定要体现在新 demo）

  - 数据分散，人工导表与二次分析成本高
  - 业务方要的不是“更多图表”，而是每天最该处理的 3-5 个问题
  - 必须支持 渠道 -> 区域 -> 门店 下钻，并带同比/环比/目标对比
  - 预警必须能转任务，且任务状态能反哺经营指标（闭环）
  - 对数据准确性、时效性非常敏感（要展示“口径说明/数据新鲜度/置信

  建议你这版 demo 做 6 个页面（足够打动客户）

  - 1. 指挥总览：今日Top风险、经营健康分、任务闭环率
  - 2. 异常诊断：比如“利润率下滑”自动拆到渠道/区域/门店并给根因
  - 3. 任务编排：AI建议动作、一键派给督导/店长、设SLA
  - 4. 执行看板：进行中/超时/升级任务，含证据回传
  - 5. 复盘中心：动作前后指标变化，策略是否有效，是否沉淀SOP
  - 6. 规则与口径：阈值、预警规则、指标定义（增强客户信任）

    不同客户的“可配置场景包”（我建议作为同一产品里的模板）

  - N多寿司包：食安红线/QSC/外卖成本异常
