# Archived FirstPlan

更新时间：2026-03-20

## 1. 本轮改动目标（已与用户确认）

### 1.1 先做结构调整（全局）
- 删除 `Page Header + Subtitle` 整块。
- `Main Workspace` 字体整体下调一档，优先压缩 `h1/h2/p/按钮`，提高单屏会话可见条数。
- `Main Workspace` 每个 tab 统一为：`角色关注摘要（紧凑） + 会话流（主角）`，不再堆大模块。
- 会话流默认首屏可见更多消息卡和证据卡，减少滚动后才进入对话的问题。

### 1.2 每个主导航的“角色关注摘要”定义
- 今日工作台：今天最该关注 3 件事（保留，但更紧凑）。
- 经营分析：`Query + 结论` 合并模块，重点覆盖：
  - 目标完成（按渠道/品类：生鲜、生熟等）
  - 当前进度、环比、目标缺口
  - 区域/督导排名
  - 营销活动营收贡献与渠道占比
  - 单店盈利模型与提升拆解（可对比其他门店样本）
- 门店风险：督导区域视角 + 最坏/最好门店 + 共性问题聚类。
- 任务协同：总部派发给我的任务、我拆解派发出去的任务、督导/门店完成情况。
- 报告中心：我订阅的日报、周报、专题（强调由日常会话沉淀）。
- 定时/规则推送：已有定时推送任务 + 当前生效规则 + 命中情况。

### 1.3 右侧沉淀结果（保留三标签，内容增强）
仅保留可切换三 tab：`任务 / 提醒 / 报告`。

#### 报告 tab（重点增强）
- 卡片字段：报告标题、来源（用户创建/AI生成）、快照ID、生成时间、结论摘要。
- KPI 摘要：如周营收、同比、外卖占比、是否已验证。
- 支持证据结构：好评率/差评率、督导区域 × 平台分布、环比变化、差评门店与 QSC 问题聚类。
- 操作项：`查看图表`、`下载报告`。

#### 任务 tab
- 展示“我下发任务”和“我当前待推进任务”的执行状态。
- 典型任务：春菜上架、食安巡检等。
- 字段：责任链路、截止时间、完成进度（百分比/已完成门店数）、当前状态（待处理/处理中/已完成/超时风险）。

#### 提醒 tab
- 展示尚未处理预警，来源分两类：下级提报、Toast AI 主动发现。
- 字段：风险级别、异常指标变化（如差评率 +x%、利润率 -x%）、影响片区/门店、建议动作、触发时间。
- 强化按严重度排序的“待处理优先级”。
