## 1. 产品定位

### 1.1 一句话定义
面向连锁餐饮总部与区域管理的 **Agent as a Service CoWork 工作台**：以对话为入口，完成“取数-诊断-下发动作-追踪闭环-复盘沉淀”。

### 1.2 与传统 SaaS 的边界
- 不是“模块堆叠+手动翻报表”的后台。
- 不是“只会回答问题”的聊天机器人。
- 是“可协作、可执行、可追踪”的数字员工工作台。

### 1.3 本轮 Demo 要验证
- 自然语言取数可替代大量人工导表。
- AI 输出以“结论+原因+动作”而非只给报表。
- 任务可以从对话中直接生成并形成闭环。
- 右侧沉淀区（任务/提醒/报告）能承接真实管理动作。

## 2. 事实锚点（访谈共识）

以下是跨访谈重复出现的事实性诉求：
- 多品牌均存在“多系统拉数、人工拼表、人工分析”的高成本流程。
- 管理层更关注“主动发现问题并推送”，不希望只做被动查询。
- 对数据可信度要求高：取数层面接近或达到 100% 准确。
- 对结果表达有要求：固定格式、结论先行、可持续追问。
- 需要跨角色闭环：总部 -> 区域/督导 -> 门店，含超时升级。
- 需要行业/商圈/竞品对标来判断“异常是否真实异常”。
- 场景高度具体：菜品表现、差评归因、目标达成、外卖到手率、会员贡献、工单时效等。

### 2.1 本轮新增访谈吸收（`qingshao.md`）
- 事实：
  - 企业微信当前无法稳定放开群聊数据读取，导致仍需人工复制内容后再喂给 AI。
  - 受访者已在真实业务中使用 AI 做财务深度分析、评价归因、报告生成，并反馈“结论质量高于人工常规分析”。
  - 连锁老板核心目标是“控制中台编制”，优先替代重复性汇总、追踪、提醒工作。
  - 高价值闭环是“发现问题 -> 直接转任务 -> 跟踪完成 -> 按周期复盘改善效果”。
- 推断：
  - 如果没有“经营接口/会员接口自动拉数 + IM 触达”，产品会被视为“可用但不够顺手”的辅助工具，而非日常中枢。
  - “专家模式（面向业务术语）”比“技能插件术语（面向技术概念）”更利于老板与管理者快速采用。
- Mock建议：
  - 在 Demo 中加入“油烟清洗/门店维保”端到端任务链（含供应商协同）和“评价共性问题立项”两个短链路。

## 3. 共性需求地图（按能力）

### 3.1 自然语言取数（P0）
- 支持跨维度：时间、渠道、区域、门店、菜品、会员、评价。
- 支持固定口径查询：同比、环比、目标达成、异常阈值。
- 支持高频模板化问题（标准问），保证结果稳定可复用。
- 支持 API 直连数据与对话取数并存（无需每次手工喂报表）。

### 3.2 主动预警与订阅（P0）
- 可配置阈值（如连续2天下滑、差评率上升、目标偏差）。
- 可配置推送对象与时间（总部、区域、督导、企微/工作台）。
- 命中后消息需附“原因摘要+建议动作”。
- 支持“次日开工即提醒”的运营节奏（如维保报告夜间入库、早晨推送风险结论）。

### 3.3 经营诊断与根因解释（P0）
- 诊断链路：指标异常 -> 渠道/区域/门店拆解 -> 根因候选 -> 动作建议。
- 明确区分“确定性数据结论”与“AI建议”。
- 支持行业/商圈/竞品对标，过滤外部波动噪声。

### 3.4 任务编排与闭环（P0）
- 对话内一键生成任务（督导/区域/门店/跨部门）。
- 任务有责任人、SLA、状态、催办、升级规则。
- 闭环后自动回看指标变化，沉淀 SOP/案例。
- 支持“企业内角色 + 外部协同方（供应商）”的混合任务链。

### 3.5 报告沉淀与稳定输出（P1）
- 自动生成日报/周报/月报，且格式稳定。
- 支持按品牌/角色拆分视图与订阅。
- 报告可回溯到触发问题与执行结果。
- 支持一键沉淀为可汇报材料（网页/PPT骨架），减少管理层二次加工。

### 3.6 低风险代操作（P1）
- 仅覆盖低风险、标准化动作（如模板化配置、批量操作）。
- 高风险动作必须双确认或人工审批。
- 全链路审计可追踪。

### 3.7 专家模式与提问引导（P1）
- 用“经营专家/维保专家/财务专家”等业务入口替代“技能列表”暴露方式。
- 在用户“不知道怎么问”时，先给可执行问题模板，再进入自由追问。

## 4. 角色需求差异

### 4.1 总部管理层 / CEO
- 看总盘：营收、利润、目标进度、门店分层、渠道结构。
- 看异常：哪些区域/门店偏离目标，是否需专项介入。
- 看决策：行业对标、资源投放、策略优先级。
- 看战略跟踪：客单价、翻台率、桌均人数等阶段性战略指标是否按月改善。

### 4.2 运营负责人 / 区域负责人
- 看执行：周维度目标达成、异常门店、重点门店提升计划。
- 看品类：新品、招牌菜、渠道活动效果与 ROI。
- 看协同：跨部门动作拆解与责任跟踪。
- 看共性痛点：评价聚类后是否已形成整改项目并落到门店动作。

### 4.3 督导
- 看清单：今日高风险门店、差评原因聚类、巡检优先级。
- 看动作：明确该做什么、先做什么、何时完成。
- 看复盘：整改后指标是否回正。

### 4.4 门店
- 看任务：可执行且具体的动作，不是抽象分析。
- 看反馈：当日关键问题、平台评价、重点改进点。

### 4.5 运维/客服/维保
- 看工单：AI先拦截常见问题，失败后自动转人工并保留上下文。
- 看效率：响应时效与解决质量提升，而非单纯降人力。
- 看闭环：周期任务（如清洗/巡检）是否按计划完成并上传合规证据。

## 5. 数据策略与约束

### 5.1 数据接入优先级
- P0 接入：美团核心经营数据（POS/外卖/团购/评价/督导执行）。
- P1 接入：会员、供应链、财务、外部平台（按品牌可用性推进）。
- P1 接入：企业 IM/OA（企微/飞书/钉钉）消息与任务回执。
- P2 接入：行业/商圈/竞品深度数据与舆情。

### 5.2 关键约束
- 数据覆盖不完整时需显式提示“覆盖率/缺口”。
- 取数与计算分层：取数追求确定性，分析标注置信度。
- 安全合规优先：账号、权限、数据托管、操作审计。
- 外部平台权限受限时，需提供“手工上传/转发”的可用兜底链路，避免流程中断。

### 5.3 口径治理
- 固定指标字典（名称、口径、周期、可比维度）。
- 报告模板稳定，避免“同问题不同输出格式”。

## 6. 新 Demo 信息架构（AaaS CoWork）

### 6.1 中心区：会话即主流程
- 每轮输出固定结构：
  1. 结论
  2. 关键指标与异常点
  3. 根因解释
  4. 建议动作
  5. 可执行按钮（建任务/订阅/生成报告）

### 6.2 右侧：沉淀区只保留三件事
- 任务
- 提醒
- 报告

### 6.3 左侧：上下文入口
- 角色视角会话（总部/区域/督导/运维）
- 历史专题会话（经营异常、菜品优化、工单提效）

## 7. 对话式 Demo 场景（用于演示）

## 7.1 场景 A：总部看昨日经营并追问异常（P0）
- Query（改写自访谈）：
  - “给我昨天全国门店经营结论，只要最重要的 5 条。”
  - “华东利润率为什么下滑？拆到门店和渠道。”
- 期望：
  - 先结论后证据；
  - 支持继续追问并转任务。

## 7.2 场景 B：评价洞察到菜品动作（P0）
- Query：
  - “看 3 月西安区域青花椒鱼差评明细、占比和高频门店。”
  - “按温度/口味/分量聚类，并生成下周督导动作。”
- 期望：
  - 区域+菜品双颗粒度；
  - 输出门店级行动建议。

## 7.3 场景 C：主动预警与订阅（P0）
- Query：
  - “建规则：连续两天外卖下滑>15%且差评率上升，10点推送给我和区域经理。”
- 期望：
  - 规则回显、确认创建、命中样例可回看。

## 7.4 场景 D：运维工单协同（P1）
- Query：
  - “今天哪些工单可以 AI 先处理？失败后自动转人工并附上下文。”
- 期望：
  - 工单不跳过，完整留痕；
  - 展示时效提升。

## 7.5 场景 E：会员/渠道结构分析（P1）
- Query：
  - “上周会员在堂食/外卖/团购/抖音占比和环比变化是什么？”
- 期望：
  - 固定口径，支持切区域/新老会员。

## 7.6 场景 F：维保周期任务自动闭环（P0）
- Query：
  - “把本月油烟清洗计划跑起来：提前通知门店和供应商，完成后校验报告，次日 9 点推送风险汇总。”
- 期望：
  - 任务可分发到门店与供应商双角色；
  - 报告异常自动标红并触发复查任务；
  - 次日推送结论+门店清单+建议动作。

## 7.7 场景 G：评价共性问题立项（P0）
- Query：
  - “把最近 30 天所有门店评价做共性归因，告诉我最该优先解决的一件事，并生成整改项目。”
- 期望：
  - 共性问题按影响度排序，而非只给零散评论；
  - 区分“可接受吐槽”与“会伤害复购/口碑”的高风险问题；
  - 可一键转 30 天整改任务并设置回看节点。

## 9. 优先级与里程碑

### P0（当前 Demo 必须具备）
- 自然语言取数（含跨维度筛选）
- 异常诊断（结论+原因+动作）
- 任务闭环（创建/催办/升级/复盘）
- 主动预警订阅
- 稳定报告骨架（可生成可回看）
- 维保/运营等周期性任务的端到端自动闭环样例

### P1（下一阶段）
- 低风险代操作
- 行业/商圈/竞品深度对标
- 运维工单 AI 拦截与协同
- 多平台舆情联动
- 专家模式与引导式提问入口
- IM 深度双向联动（任务回执、会话上下文自动注入）

### P2（探索）
- 更强预测能力（需求/备货/事件驱动）
- 跨品牌策略库与自动策略编排

## 10. 验收标准（Demo级）

- 可看：核心链路 5 分钟内讲清楚。
- 可问：每个场景至少支持 2 轮追问。
- 可做：至少能在对话中落一条任务并展示状态变化。
- 可信：明确标识事实、推断与 Mock 数据边界。
