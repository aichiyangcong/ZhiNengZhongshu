# 区域运营负责人 Query 与会话流 Mock（基于报表样例）

更新时间：2026-03-20

## 1. 目标与边界

本文件用于替换当前偏“午市补货”风格的会话主题，改为更贴近区域运营负责人日常看的报表与工作流。

数据来源依据：
- 图 1：`3月周三第二斤半价活动名单`
- 图 2：`线上平台好评率%&差评`
- 图 3：`3月业绩看板`
- 图 4：`2026年3月营业目标情况追踪`
- 文件：`docs/interview/单店盈利提升方案.xlsx`

输出形式：
- 每个场景明确归属主导航 tab
- 给出用户常问 Query
- 给出期望回复结构（结论→指标→证据→动作→沉淀）
- 提供可直接用于 Demo 的多轮会话流 mock

## 2. 区域运营负责人高频 Query（建议放入推荐问法）

### 今日工作台（home）
1. 今天目标达成最危险的 3 个区域/督导是谁？缺口各有多少？
2. 线下和线上的完成进度分别是多少？离保底线还差多少？
3. 本周活动执行次数和单均执行效果最差的是哪几个负责人？

### 经营分析（analysis）
4. 按渠道（堂食/外卖/团购）和品类（生鲜/生熟）看本月达成进度、环比和缺口。
5. 3月活动带来的营收贡献有多少？是否稀释了利润率？
6. 给我一个单店盈利提升方案，优先参考同商圈样板店。

### 门店风险（risk）
7. 哪些门店差评在上升？主要集中在哪些 QSC 问题？
8. 平台（美团/饿了么/点评/抖音）里，哪个平台拖累了综合好评率？

### 任务协同（tasks）
9. 总部下发任务里，哪些我还没拆解完？哪些督导执行掉队？
10. 春菜上架和食安巡检当前完成率是多少？超时风险在哪些门店？

### 报告中心（reports）
11. 给我输出一份周营收分析 + 好差评分析报告，带图表和下载。
12. 我订阅的日报/周报/专题，下次推送时间分别是什么？

### 定时/规则推送（push）
13. 当前规则命中里最需要我马上处理的是哪几条？
14. 下级提报和 AI 主动发现的提醒比例分别是多少？

## 3. 场景 1：3月目标追踪与达成缺口（图3+图4）

- 归属主导航：`今日工作台（home）`
- 会话主题：`3月目标追踪：渠道/品类/区域缺口`

### 用户 Query（示例）
> 今天先告诉我：3月目标还差多少，线下和线上分别差多少，最危险的三个负责人是谁？

### 期望回复（结构）
1. 结论：总体达成率 + 当前缺口金额
2. 指标：线上/线下达成率、日均目标完成、环比变化
3. 证据：负责人/区域排名，落后 Top3
4. 动作：当日优先动作（例如活动调整、督导催办、品类策略）
5. 沉淀：生成《目标追踪日报》并可下载

### Mock 会话流
- User：今天目标进度先给结论。
- Toast AI：结论先说：截至今天，3月总目标达成约 50%，总体缺口约 2.39M。线上完成快于线下，线下缺口压力更大。
  - 卡片：`summary`（结论）
  - 卡片：`kpis`
    - 总达成率：50.23%
    - 线上达成率：54.16%
    - 线下达成率：48%~49%
    - 累计缺口：-2.39M
- User：最危险的负责人和区域给我。
- Toast AI：目前掉队最明显的是苏州南区、南京北区和三督导片区，三者合计占缺口约 41%。
  - 卡片：`table`（负责人/区域/完成率/缺口/排名）
  - 卡片：`bars`（Top N 缺口排行）
- User：给今天动作。
- Toast AI：建议今天先做三件事：
  1) 线下低完成门店追加晚高峰引流动作；
  2) 外卖低转化门店复核投流和套餐；
  3) 对三名掉队负责人下发 24h 追赶任务。
  - 卡片：`task`（任务包）
  - 操作：`生成任务` / `生成目标追踪日报` / `下载报告`

## 4. 场景 2：周三第二斤半价活动复盘（图1）

- 归属主导航：`经营分析（analysis）`
- 会话主题：`活动营收贡献与利润稀释评估`

### 用户 Query（示例）
> 周三第二斤半价活动到底带来了多少营收？有没有把利润打薄？按区域和负责人拆开。

### 期望回复（结构）
1. 结论：活动是否有效（增收是否覆盖折扣）
2. 指标：活动营收、执行次数、执行均价、TC/AC/营收环比
3. 证据：区域/负责人分层对比，最佳与最差样本
4. 动作：保留/收缩/优化哪些活动规则
5. 沉淀：生成《活动效果分析》专题并可下载

### Mock 会话流
- User：活动先给结论。
- Toast AI：活动带来营收拉动，但在两片区出现“执行次数高、利润贡献低”的稀释问题。建议保留头部样板区，收缩低效区。
  - 卡片：`summary`
- User：给我拆到负责人。
- Toast AI：已按负责人拆分：A/B 两位负责人活动执行效率最高；C/D 负责人执行次数高但周营收环比为负。
  - 卡片：`table`（负责人/执行次数/周营收/周TC/周AC/环比）
  - 卡片：`bars`（执行效率 = 营收提升/执行次数）
- User：给动作。
- Toast AI：建议本周做 A/B 测试：
  1) 低效区改“满减+加价购”；
  2) 高效区保留第二斤半价；
  3) 48 小时复盘利润率。
  - 卡片：`task`
  - 操作：`生成运营整改任务` / `保存为报告` / `下载报告`

## 5. 场景 3：线上平台好评率与差评归因（图2）

- 归属主导航：`门店风险（risk）`
- 会话主题：`平台口碑风险：好评率、差评、QSC 归因`

### 用户 Query（示例）
> 线上平台好评率和差评分别是谁在拖后腿？差评主要集中在哪些门店、哪些 QSC 项？

### 期望回复（结构）
1. 结论：综合好评率变化 + 主要拖累平台
2. 指标：平台维度总评价数/好评率/差评数、环比变化
3. 证据：区域/负责人/门店明细 + QSC 聚类（出餐、打包、卫生、服务）
4. 动作：门店整改任务、督导巡检计划、平台回复策略
5. 沉淀：生成《好差评率及分析报告》（可查看图表+下载）

### Mock 会话流
- User：先告诉我哪个平台拖后腿。
- Toast AI：目前拖累综合好评率的是点评与抖音的部分片区门店，综合好评率较上月虽提升，但差评集中度上升。
  - 卡片：`summary`
  - 卡片：`kpis`（综合好评率、环比、差评门店数）
- User：门店和 QSC 细分给我。
- Toast AI：差评高发门店集中在苏州园区店、南京中山店，QSC 主要是出餐慢、打包漏液、食安记录不规范。
  - 卡片：`table`（门店/平台/好评率/差评数/QSC）
  - 卡片：`bars`（QSC 问题占比）
- User：给后续动作。
- Toast AI：已生成两类任务：
  1) 24h 差评回复与复盘任务；
  2) 48h 食安巡检任务；
  并附下次回看指标。
  - 卡片：`task`
  - 操作：`加入巡检计划` / `推送督导通知` / `下载报告`

## 6. 场景 4：单店盈利提升方案（xlsx）

- 归属主导航：`经营分析（analysis）`（生成方案） + `任务协同（tasks）`（执行） + `报告中心（reports）`（沉淀）
- 会话主题：`单店盈利提升：90天方案拆解`

### 用户 Query（示例）
> 参考 `单店盈利提升方案.xlsx`，给我做一版可执行的 90 天单店盈利提升方案，先针对粤饺皇（佳兆业店）。

### 期望回复（结构）
1. 结论：当前盈利状态、保本线、90 天目标可达性
2. 指标：成本结构、近 6 月营收/利润、渠道与品类占比、关键机会点
3. 证据：方案投入产出测算（ROI）、同类门店对标
4. 动作：分阶段任务（线下引流、外卖投流、团购优化）
5. 沉淀：形成《单店盈利提升方案》报告 + 执行任务包

### Mock 会话流
- User：先评估这家店能不能做到月均 85,000。
- Toast AI：结论：可达，但必须同步提升线下熟食客单与外卖单量。当前营业实收约 81,000，接近保本线 85,500，利润波动明显。
  - 卡片：`summary`
  - 卡片：`kpis`（当前月营收、保本线、近 6 月利润趋势）
- User：按 90 天拆动作。
- Toast AI：已拆成三阶段：
  1) 第 1-2 周：线下套餐与社群引流；
  2) 第 3-6 周：外卖投流与客单保护；
  3) 第 7-12 周：团购结构优化与复购触达。
  - 卡片：`task`（阶段目标/责任人/截止）
  - 卡片：`table`（投入、预计产出、ROI）
- User：怎么跟踪是否有效？
- Toast AI：我会每周自动回看四个指标：日均营收、外卖单量、熟食客单、月利润；若连续两周未达标，自动触发策略调整提醒。
  - 卡片：`agent`（回看编排）
  - 操作：`生成任务` / `保存为报告` / `下载报告`

## 7. 场景 5：任务闭环与推送编排

### 5.1 任务闭环（tasks）
- Query：总部下发任务我拆解后，各督导和门店完成情况怎么样？
- 期望回复：按“总部→区域→督导→门店”责任链路展示完成率、超时风险、待回看指标。
- 会话动作：`标记处理中` / `查看回看` / `生成周报素材`

### 5.2 定时/规则推送（push）
- Query：哪些提醒是下级提报，哪些是 AI 主动发现？优先级怎么排？
- 期望回复：来源拆分 + 严重度排序 + 一键推送策略（实时/汇总/周会前）。
- 会话动作：`查看规则详情` / `立即推送` / `保存为模板`

## 9. 可直接映射到右侧“沉淀结果”字段

### 报告 tab
- 标题、来源、快照 ID、生成时间、结论摘要
- KPI：周营收、同比、外卖占比、综合好评率、差评率环比
- 证据：平台分布、区域/督导排名、QSC 聚类
- 操作：查看图表、下载报告

### 任务 tab
- 任务名称、责任链路、截止时间、完成进度、当前状态
- 补充：超时风险、回看节点、关联报告

### 提醒 tab
- 严重度、来源（下级提报/Toast AI）、触发时间、异常指标、建议动作
- 排序：按严重度与影响金额/影响门店数
